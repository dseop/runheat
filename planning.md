# FitDash 기획 문서

이 문서는 제품 방향과 사용자 흐름을 빠르게 이해하기 위한 문서입니다.
화면의 정확한 규칙과 데이터 정책은 `docs/requirements.md`를 우선합니다.

## 1. 제품 개요

- 제품명: FitDash mini
- 플랫폼: iOS
- UI: SwiftUI
- 핵심 데이터: HealthKit의 러닝 워크아웃
- 핵심 화면: 연간 러닝 히트맵

앱은 사용자의 러닝 기록을 연도 단위로 한눈에 보여주고, 날짜별 세부 세션을 즉시 확인할 수 있게 하는 건강 데이터 대시보드입니다.

## 2. 제품 목표

- 러닝 기록을 날짜/연도 단위로 빠르게 탐색할 수 있게 한다.
- 별도 화면 이동 없이 히트맵 안에서 주요 정보를 확인하게 한다.
- HealthKit 러닝 데이터만 사용해 단순하고 일관된 경험을 제공한다.
- 시뮬레이터와 개발 환경에서도 데이터 확인이 가능하도록 목 데이터를 지원한다.

## 3. 사용자 시나리오

1. 사용자가 앱을 실행한다.
2. 앱이 HealthKit 읽기 권한을 요청한다.
3. 권한이 허용되면 러닝 데이터가 로딩된다.
4. 사용자는 연도 탭으로 탐색 범위를 바꾼다.
5. 사용자는 히트맵 셀을 탭해 해당 날짜의 러닝 세션을 확인한다.
6. 필요하면 다른 날짜를 연속해서 탭해 비교한다.
7. 필요하면 연 합계, 월 합계, 주 합계를 눌러 해당 기간의 히트맵 이미지를 미리본 뒤 시스템 공유 시트로 공유한다.

## 4. 핵심 화면

- 히트맵 메인 화면: 앱 실행 직후 노출되는 기본 탐색 화면
- 날짜 상세 오버레이: 셀 탭 시 확인하는 경량 상세 카드
- 공유 미리보기 시트: 선택한 연/월/주의 히트맵 이미지를 확인하고 공유하는 보조 시트

## 5. 제품 원칙

- 핵심 경험은 히트맵 안에서 끝난다.
- 별도 화면 전환은 최소화한다.
- 시뮬레이터에서도 개발용 목 데이터를 통해 화면 검증이 가능해야 한다.
- 공유도 현재 탐색 중인 기간 문맥 안에서 바로 이어져야 하며, 스크린샷 의존 대신 일관된 카드 렌더링을 사용한다.

## 6. 참조

- 정확한 화면 규칙과 데이터 정책은 `docs/requirements.md`를 따른다.
- 코드 구조와 유지보수 포인트는 `docs/development.md`를 따른다.
