# FitDash 기획 문서

이 문서는 `docs/requirements.md`를 바탕으로 정리한 제품 기획 문서입니다.  
앱의 목적, 사용자 흐름, 화면 구성, 데이터 정책을 빠르게 이해하기 위한 기준 문서로 사용합니다.

## 1. 제품 개요

- 제품명: FitDash mini
- 플랫폼: iOS
- UI: SwiftUI
- 핵심 데이터: HealthKit의 러닝 워크아웃
- 핵심 화면: 연간 러닝 히트맵

앱은 사용자의 러닝 기록을 연도 단위로 한눈에 보여주고, 날짜별 세부 세션을 즉시 확인할 수 있게 하는 건강 데이터 대시보드입니다.

## 2. 제품 목표

- 러닝 기록을 날짜/연도 단위로 빠르게 탐색할 수 있게 한다.
- 별도 화면 이동 없이 히트맵 안에서 주요 정보를 확인하게 한다.
- HealthKit 러닝 데이터만 사용해 단순하고 일관된 경험을 제공한다.
- 시뮬레이터와 개발 환경에서도 데이터 확인이 가능하도록 목 데이터를 지원한다.

## 3. 사용자 시나리오

1. 사용자가 앱을 실행한다.
2. 앱이 HealthKit 읽기 권한을 요청한다.
3. 권한이 허용되면 러닝 데이터가 로딩된다.
4. 사용자는 연도 탭으로 탐색 범위를 바꾼다.
5. 사용자는 히트맵 셀을 탭해 해당 날짜의 러닝 세션을 확인한다.
6. 필요하면 다른 날짜를 연속해서 탭해 비교한다.

## 4. 핵심 화면

### 4.1 히트맵 메인 화면

- 앱 실행 시 기본으로 노출되는 화면이다.
- 연도 선택 탭을 제공한다.
- 연간 히트맵을 세로 스크롤로 탐색한다.
- 각 날짜는 거리 기준 색상으로 표현한다.
- 오른쪽에는 주 합계를 표시한다.
- 상단에는 선택 연도의 누적 거리와 상태를 보여준다.

### 4.2 날짜 상세 오버레이

- 히트맵 셀 탭 시 노출되는 경량 카드다.
- 선택 날짜의 세션 목록을 보여준다.
- 세션별 시작 시각, 거리, 페이스를 제공한다.
- 총 거리와 세션 수도 함께 표시한다.
- 다른 날짜를 누르면 내용을 즉시 전환한다.
- 스크롤이 시작되면 오버레이는 사라진다.

## 5. 데이터 정책

- HealthKit 권한은 읽기 전용으로 `HKObjectType.workoutType()`만 요청한다.
- 조회 대상은 러닝 워크아웃만 사용한다.
- 일자 집계는 `RunDay` 중심으로 구성한다.
- 세션 단위 데이터는 `RunSession` 모델로 다룬다.
- 연도별 화면 데이터는 캐시 우선 정책을 따른다.
- 시뮬레이터에서는 `MockHealthKitManager`와 `Resources/*.json`을 사용한다.

## 6. 범위

### 포함

- 연간 히트맵
- 연도 탭
- 날짜별 상세 오버레이
- 러닝 세션 목록 표시
- 연도별 합계 및 주 합계
- 개발용 Mock 데이터

### 제외

- 별도 List 화면
- 별도 Calendar 화면
- 별도 상세 푸시 화면
- HealthKit 쓰기 권한
- 러닝 외 운동 타입

## 7. 운영 원칙

- 초기 화면은 히트맵만 유지한다.
- 인증 로직은 중복 호출되지 않도록 단일 진입점에서 관리한다.
- 개발 전용 기능은 `#if DEBUG` 범위로 제한한다.
- 문서 기준이 바뀌면 요구사항 문서와 함께 갱신한다.

## 8. 성공 기준

- 앱 실행 후 곧바로 히트맵이 보여야 한다.
- 사용자가 특정 연도와 날짜를 빠르게 탐색할 수 있어야 한다.
- 세션 상세 정보가 히트맵 내에서 즉시 보여야 한다.
- 시뮬레이터에서도 화면 검증이 가능해야 한다.
